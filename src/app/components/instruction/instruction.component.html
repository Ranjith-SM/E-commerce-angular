Steps used and challenges faced to build this application:-
  1. This Project consists of multiple components such as Home, Cart, Login, Register and Orders and for the navigation between components Routing has been used via app routing module by providing respective routes in the route array.
  2. The respective service and models required for the application has been generated using the commands in git bash
  3. Used Template Driven form validation for the login and register page.
  4. For fetching the products an Api call has been made using HTTPClient (GET) request.
  5. Stored the product, cart and order details in the Local Storage . Stored the user details in the Session Storage.
  6. The add to cart and orders functionality logic has been designed in a logic that each user can access only his/her cart and order details in his/her login.
  7. No logged in user can access cart order and log out so for that CanActivate() method of auth guard in angular. For this process session storage has been used.
  8. Every Authenticated user can access cart to add or remove product also check out product to see it the orders page.
Challenges:
  • Frequent API call - Used Observer() method to reduce the API call by fetching and checking it with the local storage.
  • Handling Cart for individual user.
